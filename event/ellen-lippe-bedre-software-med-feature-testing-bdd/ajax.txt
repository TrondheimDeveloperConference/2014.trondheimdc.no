<a class="pops-exit" href="http://2014.trondheimdc.no/">&times;</a><img width="400" height="400" src="http://2014.trondheimdc.no/wp-content/uploads/2014/08/image-400x400.jpg" class="pops-face wp-post-image" alt="Ellen Lippe" /><h2> Bedre software med feature testing (BDD):</h2><h4>Klokken 12:30, spor 4</h4><p>Automatisert feature testing (BDD) kan muliggjøre hyppig leveranse av software med god kvalitet. Selv om man er enig i dette, kan det ofte være vanskelig å gjennomføre i praksis. I denne presentasjonen vil jeg dele mine erfaringer med BDD fra flere team i Statoil, og komme med tips om hva jeg mener er nødvdendig for å lykkes med BDD. Presentasjonen vil ta for seg tankesett, arkitektur og kultur.</p>
<h4>ELLEN LIPPE:</h4><p>IT-utdanning fra NTNU, jobbet som software developer i Statoil siden 2002. Har stor tro på at et tankesett inspirert av domain driven design (DDD) og agile prinsipper, som f.eks BDD, er nøkkelen for å kunne lage kompleks, langlevd software som kan endres kontinuerlig i takt med brukernes behov</p>
